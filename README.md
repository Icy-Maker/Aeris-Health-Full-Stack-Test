## 任务A：能量波总能量计算
### 功能说明
根据输入的单道能量波基础数值（x）和能量波总道数（n），按照以下规则计算总能量值：
- 若能量波总道数 `n` 为奇数，总能量值等于单道能量波基础数值 `x`；
- 若能量波总道数 `n` 为偶数，总能量值为 0；
- 若仅输入一个数值，默认 `n=0`（偶数），总能量值为 0。

### 使用方法
1. 运行脚本：
   ```bash
   python task_A.py
   ```
2. 按照提示输入参数，支持三种分隔方式：
   - 空格分隔（例：`10 5`）
   - 英文逗号分隔（例：`10,5`）
   - 中文逗号分隔（例：`10，5`）
3. 脚本自动输出计算后的总能量值。

### 代码核心说明
| 函数 | 作用 |
|------|------|
| `calc_total_energy(x: int, n: int) -> int` | 核心计算函数，接收基础值 `x` 和总道数 `n`，返回总能量值 |
| `main()` | 处理用户输入（兼容多分隔符）、调用核心函数并输出结果 |

### 运行示例
示例1：输入奇数道数
```
请输入能量波数值和能量波道数：10 5
10
```

示例2：输入偶数道数
```
请输入能量波数值和能量波道数：20 4
0
```

示例3：仅输入单个数值
```
请输入能量波数值和能量波道数：30
0
```

## 任务B：飞船推进器配置数量计算
### 功能说明
根据输入的推进器总数（n），计算合法配置下的**最少飞船数**和**最多飞船数**，规则如下：
- 合法条件：推进器总数 `n ≥ 4` 且为偶数；不满足则输出 `-1`；
- 最多飞船数：优先使用A型飞船（每艘配4个推进器），即 `n // 4`；
- 最少飞船数：优先使用B型飞船（每艘配6个推进器），无法整除时向上取整。

### 使用方法
1. 运行脚本：
   ```bash
   python task_B.py
   ```
2. 按照提示输入推进器总数（整数）；
3. 脚本输出结果：
   - 合法配置：输出最少/最多飞船数；
   - 非法配置：输出 `-1`。

### 代码核心说明
| 函数 | 作用 |
|------|------|
| `calc_craft_number(n: int)` | 核心计算函数，合法时返回 `(最少飞船数, 最多飞船数)`，无解返回 `None` |
| `main()` | 处理用户输入、调用核心函数并输出结果 |

### 运行示例
示例1：合法输入（n=12）
```
请输入所有飞船的推进器总数：12
最少飞船数：2
最多飞船数：3
```

示例2：合法输入（n=14）
```
请输入所有飞船的推进器总数：14
最少飞船数：3
最多飞船数：3
```

示例3：非法输入（n=3，小于4且奇数）
```
请输入所有飞船的推进器总数：3
-1
```

示例4：非法输入（n=5，奇数）
```
请输入所有飞船的推进器总数：5
-1
```

## 任务D：电商商品详情页（Mock API）

## 项目概述

基于纯 HTML/CSS/JavaScript 实现的电商商品详情页，支持商品展示、变体切换、数量控制、加入购物车等完整交互流程。所有数据通过前端 Mock API 模拟，无需真实后端。

## 技术栈

- HTML5
- CSS3（Flexbox + Grid + 响应式）
- Vanilla JavaScript（ES6+）
- 无框架依赖

## 功能特性

| 功能 | 说明 |
|:---|:---|
| 商品切换 | 4件商品循环切换（耳机、听诊器、围巾、手提包） |
| SKU 变体切换 | 同一商品下不同规格切换，图片/价格/库存同步更新 |
| 数量控制 | ±按钮调整，边界限制为 1 ~ 可用库存 |
| 库存实时显示 | 展示当前可用库存 / 总库存 |
| 加入购物车 | 成功后购物车角标累加，库存实时扣减 |
| 缺货处理 | 库存为0时禁用加购，提示"暂时缺货" |
| 错误反馈 | 库存不足、网络异常等场景友好提示 |

## 运行方式

### 方案A：本地运行（推荐）

需安装 VS Code + Live Server 插件，支持完整交互与热更新。

**步骤：**

1. 安装 [VS Code](https://code.visualstudio.com/)
2. 安装 Live Server 插件（Extensions 搜索 "Live Server" → Install）
3. 用 VS Code 打开 `task_D` 文件夹
4. 右键 `index.html` → **Open with Live Server**
5. 浏览器自动打开 `http://127.0.0.1:5500/`，即可交互

**验证 Live Server 是否安装成功：**

右键 `test.html` → Open with Live Server，若浏览器显示"Live Server 运行正常"，则安装成功。验证完成后无需重复测试。

---

### 方案B：直接打开 HTML 文件（有限功能）

无需安装任何工具，但部分功能可能受限。

**步骤：**

1. 进入 `task_D` 文件夹
2. 双击 `index.html` 用系统默认浏览器打开
3. 基础展示功能可用，但以下特性可能异常：
   - ES Module 模块化加载（若浏览器不支持）
   - 本地图片跨域（部分浏览器安全限制）
   - 热更新（需手动刷新）

> **建议**：优先使用方案A，确保所有功能正常。

## 状态说明

| 页面状态 | 触发条件 | 展示效果 |
|:---|:---|:---|
| LOADING | 页面加载/切换商品 | 中央旋转动画 + "正在加载" |
| SUCCESS | 商品数据加载成功 | 完整商品信息，可交互 |
| OUT_OF_STOCK | 当前 SKU 库存为 0 | 加购按钮禁用，显示"暂时缺货" |
| ADDING | 点击加购后请求中 | 按钮显示"添加中..."，禁用点击 |
| ADD_SUCCESS | 加购成功 | 绿色提示"已添加X件到购物车"，角标更新 |
| ADD_FAILURE | 加购失败 | 红色提示错误原因（如库存不足） |
| API_ERROR | 数据加载失败 | 错误信息 + "重新加载"按钮 |

## 注意事项

1. **购物车数据为内存存储**，刷新页面后重置
2. **库存为实时计算**：`可用库存 = 总库存 - 购物车已占用`
3. 首次加载默认展示 **P001 无线降噪耳机 Pro**
4. 所有价格单位为美元（$），SKU 字体使用 Times New Roman
5. 图片缺失时显示渐变占位背景，不影响功能操作
